# Implementation Plan Template

## Purpose

Translate an approved specification into an implementation strategy.

## When to Use

Use this template after the specification and any required ADRs are approved.

## Expected Sections

1. Title
2. Related Specification
3. Implementation Strategy
4. Affected Areas
5. Milestones
6. Validation Strategy
7. Rollback or Recovery Notes
8. Assumptions

## Completion Criteria

- The path from specification to implementation is clear.
- Affected repository areas are identified.
- Validation is defined before coding starts.
- Assumptions and constraints are documented.
