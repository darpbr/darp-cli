# Architecture Decision Record Template

## Purpose

Capture an architectural decision and the reasoning behind it.

## When to Use

Use this template when a specification requires a meaningful technical decision with tradeoffs.

## Expected Sections

1. Title
2. Status
3. Context
4. Decision
5. Alternatives Considered
6. Consequences
7. Follow-Up Actions

## Completion Criteria

- The decision is explicit.
- Tradeoffs are documented.
- Consequences are understandable to future contributors.
- The record links back to the relevant specification.
