# Specification Template

## Purpose

Describe a feature before any implementation starts.

## When to Use

Use this template for every approved feature or substantial change.

## Expected Sections

1. Title
2. Status
3. Context
4. Problem Statement
5. Goals
6. Non-Goals
7. Requirements
8. User Flows
9. Acceptance Criteria
10. Risks and Open Questions

## Completion Criteria

- The problem is clearly defined.
- Scope boundaries are explicit.
- Requirements are testable.
- Acceptance criteria can be reviewed before implementation.
